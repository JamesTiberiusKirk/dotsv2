<title>Expose getAvailableHandsets GraphQL query</title>
<description>
Add the GraphQL query for `getAvailableHandsets`.

Scope:

* Define the schema contract for the query
* Resolve request context and required inputs
* Integrate with the backend service(s)
* Return the response shape needed by the frontend
* Handle query-specific business rules and error cases

This sub-issue covers the end-to-end delivery of this operation.

**Acceptance criteria**

* A new query `mobileAvailableHandsets(serviceId: String!)` is exposed on the public GraphQL schema
* `customerId` is resolved from the authenticated context, not passed as a query argument
* The response is an array of brands, each with a name and array of phones
* Each phone has model-level fields: `model`, `fromPrice`, `screenSize`, `battery`, `wifiStandard`, `eSim`, `dualSim`, `specificationsLink`
* Each phone has an `optionTypes` array of `{ key, values }` representing the available option dimensions (for example `colour`, `storage`)
* Each phone has a `variants` array, and each variant has: `id`, `attributes: [{ key, value }]`, `monthlyCharge`, `rrp`, `inStock`
* The query accepts an optional `includeOutOfStock: Boolean` filter, which defaults to `false`
* If the service already has an active handset contract, an empty list is returned
* If the customer fails pre-eligibility checks, an appropriate error is returned
* The resolver calls `GetAvailableHandsets` on the handset gRPC service
* The `public-graphql` service establishes a gRPC connection to the handset service
</description>
